# mercenary-reselect UI layout prompt

이 프롬프트는 **용병 재선택 프로토타입 UI 수정 요청용**입니다.  
구조를 멋대로 바꾸지 않고, 내가 정한 화면 규칙과 게임형 UX 기준 안에서만 수정하게 만드는 목적입니다.

## 핵심 원칙

1. **내가 지정한 화면 범위를 먼저 따른다**
   - 전체 화면 기준은 16:9 스테이지다.
   - 스크롤은 전체 화면이 아니라, 필요한 내부 영역에서만 사용한다.
   - 인벤토리처럼 정말 필요한 경우에만 내부 스크롤을 허용한다.
   - 가능하면 한 화면에 최대한 담는다.

2. **작업 전에 구조를 멋대로 추가하지 않는다**
   - 새 화면, 새 팝업, 새 상세 패널, 새 흐름을 임의로 만들지 않는다.
   - 내가 지정한 범위 안에서만 수정한다.
   - 구조 변경이 필요하면 먼저 내가 의도한 내용을 요약해서 확인받는다.

3. **게임 UI처럼 보여야 한다**
   - 단순 웹 페이지 느낌보다 실제 게임 UX/UI 흐름을 우선한다.
   - 팝업은 중앙 팝업처럼 보여야 하고, 배경 씬과 레이어 관계가 자연스러워야 한다.
   - 뎁스가 어색하면 안 된다. 팝업, 딤, 배경, 카드의 우선순위가 명확해야 한다.

4. **텍스트는 최소화한다**
   - 상태를 슬롯/아이콘/배지/버튼 상태로 표현할 수 있으면 텍스트를 줄인다.
   - 설명 문구는 꼭 필요한 1줄만 둔다.
   - 버튼 외 불필요한 안내 문구는 함부로 추가하지 않는다.

5. **정보 밀도는 높이고, 덩어리는 줄인다**
   - 너무 넓거나 너무 높은 카드, 행, 패널을 만들지 않는다.
   - 카드형보다 보드형, 리스트형처럼 기능 위주로 정리한다.
   - 한 행에서 비교와 선택이 끝나야 한다.

## 용병 재선택 UI 규칙

1. **리스트형으로 보여준다**
   - `한 행 = 한 재선택 대상`
   - 여러 행이 세로로 반복되는 구조여야 한다.
   - 좌측 목록 / 우측 상세 패널처럼 나누지 않는다.

2. **재선택 대상은 여러 개일 수 있다고 가정한다**
   - `원본 + 후보 슬롯들 + 우측 액션`
   - 한 행 안에 모두 보여준다.

3. **전환 연출은 슬롯 오픈형**
   - 드르륵 굴리는 연출 금지
   - 빈 슬롯이 열리고, 결과가 채워지는 방식으로 처리

4. **버튼은 통일감 있게**
   - 같은 역할 계열 버튼은 크기/정렬/비율을 통일한다.
   - 우측 액션 영역은 한 덩어리처럼 보여야 한다.

5. **팝업 내 UX**
   - 팝업은 씬 전체를 덮는 화면이 아니라 중앙에 떠 있는 구조여야 한다.
   - 팝업 안 리스트는 필요 시 내부 스크롤 가능
   - 전체 화면은 스크롤하지 않는다

6. **기본 상태는 명확해야 한다**
   - 첫 화면은 소환 씬만 보여야 한다.
   - 재선택 팝업은 기본 숨김 상태여야 한다.
   - 버튼을 눌렀을 때만 팝업이 열려야 한다.

## 실패 방지 규칙

1. **레이아웃 요구사항은 코드 구조가 아니라 실제 화면 결과 기준으로 검증한다.**
2. **같은 형태의 UI는 공용 클래스/공용 규칙으로 묶고, 개별 요소마다 따로 처리하지 않는다.**
3. **구현 후에는 브라우저에서 실제로 어떻게 보이는지 확인하기 전까지 완료로 판단하지 않는다.**

## 수정 요청 방식

아래 형식으로 수정한다.

1. 먼저 현재 UI 문제를 **3개 이하로 짚는다**
2. 내가 말한 의도를 한 번 요약한다
3. 그 다음 수정한다
4. 수정 후 무엇을 바꿨는지 짧게 정리한다

## 금지 사항

- 내가 말하지 않은 새 화면 추가
- 내가 말하지 않은 새 플로우 추가
- 리스트형을 상세 패널 구조로 바꾸기
- 전체 화면을 팝업처럼 쓰기
- 버튼/카드 크기 통일성 무시
- 텍스트를 필요 이상으로 추가
- 확인 없이 디자인 해석을 바꿔버리기

## 바로 복붙용 요청문

이 HTML 파일을 기준으로 UI를 수정해줘.

규칙:
- 내가 지정한 16:9 전체 화면 기준을 먼저 따른다.
- 전체 화면은 스크롤하지 말고, 필요한 경우 내부 리스트만 스크롤한다.
- 새 화면이나 새 플로우를 임의로 만들지 말고, 먼저 내가 의도한 구조를 유지한다.
- 용병 재선택은 리스트형 구조로 보여줘. 한 행 안에 원본/후보/액션을 같이 배치해줘.
- 좌측 목록 / 우측 상세 구조로 바꾸지 마.
- 전환 연출은 드르륵 말고 빈 슬롯이 열리며 결과가 채워지는 방식으로 해줘.
- 버튼은 같은 계열끼리 크기와 비율을 통일해줘.
- 텍스트는 최소화하고, 가능하면 슬롯/아이콘/배지로 상태를 표현해줘.
- 먼저 현재 UI 문제를 3개 이하로 짚고, 내 의도를 짧게 요약한 뒤 수정해줘.
