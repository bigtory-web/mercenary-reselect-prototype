# mercenary-reselect prototype prompt

이 프로젝트의 프로토타입은 **용병 재선택 시스템의 UX 검증**이 목적이다.  
실제 서비스용 완성 코드보다, **규칙 이해 / 선택 흐름 / 화면 체감**을 먼저 검증한다.

## 목적

- 고등급 중복 결과에 대한 재선택 UX가 직관적인지 확인
- 원본 유지 구조가 체감상 자연스러운지 확인
- 랜덤 후보 누적 후 마지막 선택 흐름이 납득되는지 확인

## 핵심 규칙

- 원본 용병 유지
- 전환 기회 2회
- 같은 등급 풀 8마리
- 후보 중복 재등장 없음
- 마지막에 `원본 + 후보` 중 1개 선택

## UI 원칙

1. **리스트형 UI로 구성**
   - 카드 1장 중심이 아니라 여러 재선택 대상을 동시에 비교하는 구조로 본다.

2. **재선택 대상은 여러 개일 수 있다고 가정**
   - 최소 `원본 + 후보 n개`를 함께 보여줄 수 있어야 한다.

3. **전환 연출은 슬롯 오픈형**
   - 드르륵 굴러가는 연출은 쓰지 않는다.
   - 빈 슬롯이 열리고 결과가 채워지는 방식으로 처리한다.

4. **우측 액션 영역 분리**
   - `재선택 비용`
   - `최종 선택`
   버튼은 비교 영역과 분리된 액션 영역에 둔다.

5. **핵심 루프는 한 화면 안에서 끝나야 함**
   - 모바일 기준으로 스크롤 없이
   - 현재 상태 확인 → 재선택 → 최종 선택
   흐름이 한 화면에서 끝나야 한다.

6. **시스템 규칙은 바꾸지 말고 UI 구조만 조정**
   - 규칙 검증용 프로토타입이므로, 룰보다 레이아웃과 체감 개선을 우선한다.

## 요청 템플릿

이 HTML 파일을 기준으로 UI를 수정해줘.

규칙:
1. 카드 1장 중심이 아니라 리스트형 비교 UI로 바꿔줘.
2. 재선택 대상은 여러 개일 수 있으니, 원본과 후보들을 동시에 보여주는 구조로 만들어줘.
3. 전환 연출은 드르륵 굴리는 방식 말고, 빈 슬롯이 열리면서 결과가 채워지는 방식으로 해줘.
4. 재선택 비용 버튼과 최종 선택 버튼은 우측 액션 영역에 고정해줘.
5. 핵심 루프는 스크롤 없이 한 화면 안에서 끝나게 유지해줘.
6. 시스템 규칙은 바꾸지 말고 UI 구조만 바꿔줘.
