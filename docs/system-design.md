# 용병 재선택 시스템

## 기획 의도

수집형/방치형 RPG에서 고등급 용병 중복은 기본적으로 성장 자원으로 기능한다.  
다만 수집이 진행될수록, 이미 보유한 고등급 용병이 반복적으로 등장할 때 성장 가치와 별개로 감정적 허탈감이 발생할 수 있다.

본 시스템은 이러한 상황에서 결과를 즉시 확정하지 않고, 제한된 범위 안에서 다시 선택할 기회를 제공함으로써 직접 획득 가치를 유지하면서도 원하지 않는 고등급 결과에 대한 체감 손실을 완화하는 것을 목표로 한다.

## 핵심 정의

- 시스템 이름: `용병 재선택 시스템`
- 성격: `고등급 중복 결과에 대한 제한적 재선택 시스템`
- 목적:
  - 고등급 중복 허탈감 완화
  - 수집 후반 구간의 뽑기 체감 가치 보정
  - 직접 획득 가치와 재선택 기회 간 균형 유지

## 기본 규칙

1. 용병은 총 5개 등급으로 구성된다.
2. 등급 간 이전은 불가하다.
3. 전환은 상위 2개 등급에만 적용된다.
4. 최고 등급은 동일 중복 3개가 필요하다.
5. 차상위 등급은 동일 중복 1개가 필요하다.
6. 전환에는 별도 재화인 `전환기`가 필요하다.

## 재선택 플로우

1. 유저가 고등급 중복 용병을 획득한다.
2. 해당 결과는 즉시 확정되지 않고 `임시 보관함`으로 이동한다.
3. 유저는 원본 용병을 그대로 수령하거나, 전환을 진행할 수 있다.
4. 전환을 진행하면 같은 등급 풀에서 랜덤 후보가 생성된다.
5. 전환 후보는 리스트에 누적된다.
6. 전환 기회는 총 2회까지 허용된다.
7. 중복 후보는 다시 등장하지 않는다.
8. 최종적으로 유저는 `원본 + 누적 후보` 중 1개를 선택해 수령한다.
9. 최종 수령 시 전환 절차는 종료된다.

## 현재 프로토타입 기준값

- 같은 등급 후보 풀: 8마리
- 전환 기회: 2회
- 후보 중복 재등장: 없음
- UI 방식: 카드 리스트

## 검토 포인트

### 무한 전환 가능성(상)
- 검토 포인트: 전환 결과가 다시 전환 재료가 되면 반복 루프가 성립할 수 있다.
- 대응: 전환은 `임시 보관함` 상태에서만 가능하게 하고, 최종 수령 시 종료되며 재전환은 불가하게 설계한다.
- 남는 과제: 전환 기회 수가 과도하면 사실상 원하는 결과에 수렴할 수 있으므로, 풀 크기 대비 적정 기회 수 검증이 필요하다.

### 전환 가치가 직접 획득보다 커지는 경우(상)
- 검토 포인트: 직접 획득보다 중복 후 전환이 더 기대되는 구조가 되면 가챠 의미가 흔들릴 수 있다.
- 대응: 전환기 소모, 등급 제한, 중복 재료 요구 조건으로 전환 진입 비용을 높게 설계한다.
- 남는 과제: 전환기 공급량이 높아지면 체감상 전환 가치가 과도해질 수 있어 수급 설계 검증이 필요하다.

### 선택 전환이 너무 강한 경우(중)
- 검토 포인트: 완전 선택형 전환은 “등급만 맞으면 된다”는 체감으로 이어질 수 있다.
- 대응: 현재 안에서는 완전 선택 전환을 제외하고, 랜덤 후보를 축적한 뒤 마지막에 선택하는 방식으로 제한한다.
- 남는 과제: 후보 축적 수가 많아질수록 선택 전환에 가까워질 수 있으므로, 기회 수 상한 검토가 필요하다.

### 랜덤 전환의 허탈감(하)
- 검토 포인트: 원하는 결과가 후보에 뜨지 않으면 다시 허탈감이 생길 수 있다.
- 대응: 랜덤 결과를 즉시 버리지 않고 후보로 누적해 마지막에 최선의 결과를 선택하도록 설계한다.
- 남는 과제: 랜덤 후보 풀이 지나치게 넓거나 기회 수가 적으면 체감 위로가 약해질 수 있다.
