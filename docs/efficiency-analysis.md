# 소환 재선택 시스템 효율 검증

## 목적

이 문서는 `특정 SSR 1종을 목표로 과금/소환할 때`, 확정 관리 시스템이 기대 비용과 체감 가치를 얼마나 바꾸는지 비교하기 위한 검증 메모다.

- 본 문서는 시스템 본문이 아니라 **프로토타입 기반 비교 분석**을 다룬다.
- 사용 수치는 현재 프로토타입 테스트 세팅을 기준으로 한 **가정값**이다.

## 검증 가정

| 항목 | 값 | 비고 |
| --- | --- | --- |
| SSR 풀 수 | 8명 | 목표 SSR 1종 기준 |
| SSR 확률 | 5% | 1회 소환 기준 |
| 10회 소환 비용 | 보석 2000 | 프로토타입 기준 |
| 확정 관리 구조 | 원본 + 후보 2개 | 후보는 원본 제외 SSR 풀에서 생성 |
| 전환기 소모 | 재선택 1회당 1개 | 효율 비교에 참고값으로 사용 |

## 계산식

### 1. 시스템 없음

특정 SSR 1종을 직접 뽑을 확률:

`0.05 × (1 / 8) = 0.00625`

10회 소환 1회에서 목표 SSR 1종을 얻을 확률:

`1 - (1 - 0.00625)^10 ≈ 0.0608`

즉, 시스템이 없을 때 목표 SSR 1종을 얻을 10회 소환 기준 성공 확률은 약 `6.08%`다.

### 2. 시스템 있음

SSR 결과 1개가 목표 SSR로 이어질 수 있는 경우는 두 가지다.

- 원본이 이미 목표 SSR인 경우: `1 / 8`
- 원본은 아니지만 후보 2개 안에 목표 SSR이 포함된 경우: `7 / 8 × 2 / 7 = 1 / 4`

따라서 SSR 결과 1개가 목표 SSR 접근 기회가 될 확률은:

`1 / 8 + 1 / 4 = 3 / 8 = 37.5%`

이 값을 SSR 확률 5%와 곱하면, 1회 소환 결과가 목표 SSR 접근 기회가 될 확률은:

`0.05 × 0.375 = 0.01875`

10회 소환 1회에서 목표 SSR 접근 기회를 얻을 확률:

`1 - (1 - 0.01875)^10 ≈ 0.1724`

즉, 시스템이 있을 때 목표 SSR 1종을 얻거나 후속 재선택으로 접근할 수 있는 10회 소환 기준 성공 확률은 약 `17.24%`다.

## 비교 표

| 비교 항목 | 시스템 없음 | 시스템 있음 |
| --- | --- | --- |
| 목표 SSR 1종 기준 10회 소환 성공 확률 | 6.08% | 17.24% |
| 목표 달성까지 기대 10회 소환 횟수 | 16.46회 | 5.80회 |
| 목표 달성까지 기대 보석 소모량 | 약 32,910 | 약 11,598 |
| 중복 SSR의 가치 | 즉시 손실 | 재선택 후보로 전환 |
| 중복 SSR의 목표 SSR 기여도 | 없음 | SSR 1개당 37.5% 접근 기회 |

### 해석

- 현재 프로토타입 기준으로 보면, 확정 관리 시스템은 특정 SSR 1종 획득 기대 비용을 약 `2.84배` 효율화한다.
- 기대 보석 소모량 기준으로 보면 약 `64.8%` 수준의 비용 절감 효과가 생긴다.
- 이는 중복 SSR이 즉시 손실로 종료되지 않고, 후속 선택 기회로 전환되기 때문이다.

## 전환기 관점 보정

- 후보 2개 안에 목표 SSR이 포함된 경우, 목표 후보는 첫 번째 또는 두 번째 슬롯에 같은 확률로 등장한다.
- 따라서 성공적인 재선택 1건당 기대 전환기 소모량은 평균 `1개`로 볼 수 있다.
- 즉, 시스템이 기대 보석 비용을 낮추는 대신 `전환기`라는 별도 재화 압박을 만든다.

따라서 실제 효율 평가는 아래 두 축을 함께 봐야 한다.

- 보석 기준 목표 SSR 접근 비용 감소
- 전환기 기준 추가 선택 비용 발생

## 체감 가치 해석

| 상황 | 시스템 없음 | 시스템 있음 |
| --- | --- | --- |
| 중복 SSR 획득 | 즉시 손실 체감 | 후속 선택 기회로 전환 |
| 소환 종료 후 행동 | 종료 가능성 높음 | 확정 관리 진입 가능성 발생 |
| 목표 SSR 실패 시 감정 | 재화 손실 중심 | `다시 한 번 정리해볼 수 있다`는 기대 형성 |

즉, 이 시스템의 가치는 단순히 기대 비용을 낮추는 것뿐 아니라, `희귀 결과가 실패로 끝나지 않게 만드는 감정 보정 장치`라는 데 있다.

## 해석 시 주의점

- 본 비교는 현재 프로토타입의 테스트 세팅을 기준으로 한 값이다.
- 실제 라이브 서비스에서는 SSR 확률, SSR 풀 수, 후보 구조, 전환기 공급량에 따라 효율 상승폭이 달라질 수 있다.
- 특히 효율 상승폭이 너무 크면 직접 소환보다 확정 관리 가치가 더 커질 수 있으므로, 운영 기준에선 별도 상한 검토가 필요하다.
