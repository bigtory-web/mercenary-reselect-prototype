# 용병 소환 및 재선택 시스템 제출용 슬라이드 문안

## 슬라이드 1. 시스템 정의

**제목**  
용병 소환 및 재선택 시스템

**핵심 문안**
- 소환 결과 확인 이후, 고등급 중복 결과를 후속 선택 경험으로 전환하는 시스템
- 소환 자체를 대체하지 않고, 희귀 결과의 손실 체감을 완화하는 보정 장치

**보조 문장**
- 본 시스템은 `소환 → 결과 확인 → 후속 선택 → 보유 누적` 흐름을 설계한다.

**이미지 필요**
- 소환 메인 화면 1컷

---

## 슬라이드 2. 문제 정의

**제목**  
중복 SSR은 수집 후반부에 손실 경험으로 작동한다

**핵심 문안**
- 보유 슬롯이 채워질수록 원하는 결과를 얻기 어려워진다.
- 이 구간에서 중복 SSR은 성장 자산보다 `원하는 결과를 얻지 못한 실패`로 체감되기 쉽다.
- 과금 효율이 운에 과도하게 종속되면, 뽑기 지속 의사 자체가 약해질 수 있다.

**보조 문장**
- 핵심 문제는 `희귀 결과가 나와도 만족이 아니라 종료 감각으로 이어진다`는 점이다.

**이미지 필요**
- 중복 SSR이 나온 소환 결과 팝업

---

## 슬라이드 3. 시스템 역할

**제목**  
소환 결과의 감정 경험을 후속 행동으로 연결한다

**핵심 문안**
- 소환은 결과를 체감하는 메인 획득 경험을 담당한다.
- 확정 관리는 중복 SSR을 후속 의사결정 대상으로 전환한다.
- 영웅 인벤토리는 획득 결과가 실제로 누적되고 있다는 감각을 보완한다.

**보조 문장**
- 본 시스템은 독립 기능이 아니라, 메인 가챠 루프 안에서 작동하는 감정 정리 장치다.

**이미지 필요**
- 소환 화면 / 결과 팝업 / 확정 관리 / 영웅 인벤토리 연결 구조도

---

## 슬라이드 4. 핵심 구조

**제목**  
소환 → 결과 확인 → 확정 관리 → 보유 누적

**핵심 문안**
- 10회 소환 결과는 결과 팝업에서 먼저 확인한다.
- 소환 결과는 즉시 보유 현황에 반영된다.
- SSR 결과는 확정 관리 대상이 되어 후속 선택 기회를 만든다.
- 최종 선택 결과는 인벤토리에 반영되고, 관리 리스트에서는 제거된다.

**보조 문장**
- 재선택은 독립 메뉴가 아니라, 소환 결과 이후 의미가 생기는 후속 기능으로 설계했다.

**이미지 필요**
- 10회 소환 결과 팝업 / 확정 관리 팝업 2컷

---

## 슬라이드 5. 핵심 규칙

**제목**  
원본 유지형 재선택 구조

**핵심 문안**
- 확정 관리는 SSR 결과가 나왔을 때만 진입 가치가 생긴다.
- 원본 SSR은 유지된다.
- 재선택 시 후보 슬롯이 순차적으로 열린다.
- 유저는 원본 또는 열린 후보 중 하나를 최종 선택한다.
- 재선택 시도에는 전환기가 소모된다.

**보조 문장**
- 직접 획득 가치와 후속 보정 가치를 분리하기 위해, 소환과 재선택은 서로 다른 재화를 사용한다.

**이미지 필요**
- 확정 관리 팝업 1컷

---

## 슬라이드 6. 유저 행동 변화

**제목**  
중복 SSR은 손실에서 선택 기회로 바뀐다

**핵심 문안**
- 도입 전: 중복 SSR은 즉시 손실로 받아들여지고, 소환 경험은 단발성으로 끝난다.
- 도입 후: 중복 SSR은 확정 관리로 이어지며, 결과 이후의 행동이 한 단계 더 생긴다.
- 결과적으로 직접 획득 가치는 유지하면서, 손실 체감은 완화한다.

**보조 문장**
- 핵심 변화는 `아쉬운 결과`를 `정리 가능한 결과`로 재인식하게 만드는 것이다.

**이미지 필요**
- 도입 전/후 유저 행동 비교 도식

---

## 슬라이드 7. 효율 검증 관점

**제목**  
목표 SSR 접근성보다, 과금 효율 붕괴 완화가 핵심이다

**핵심 문안**
- 본 시스템은 특정 SSR을 싸게 주는 장치가 아니다.
- 중복 SSR이 즉시 손실로 끝나지 않고, 후속 재선택 기회로 전환된다.
- 따라서 동일한 소환 재화라도 `직접 획득 확률 + 후속 선택 기회`가 합쳐져, 목표 결과 접근성이 높아진다.
- 핵심 가치는 효율 상승 자체보다 `운 요소로 무너지는 과금 체감의 완충`에 있다.

**보조 문장**
- 효율이 너무 높아지면 직접 소환 가치가 흔들릴 수 있으므로, 개선보다 `붕괴 방지선 형성`에 가깝게 설계해야 한다.

**이미지 필요**
- 효율 비교 표 또는 기대 비용 비교 도식

---

## 슬라이드 8. 리스크 및 검증 포인트

**제목**  
손실 완화 장치와 직접 소환 가치의 균형이 필요하다

**핵심 문안**
- SSR 유입 빈도가 높으면 확정 관리가 보상보다 관리 피로로 느껴질 수 있다.
- 재선택 효율이 직접 획득보다 높게 느껴지면, 소환 결과보다 후속 교환이 더 중요한 구조로 오해될 수 있다.
- 반대로 재선택 체감이 약하면 손실 완화 장치로 기능하지 못한다.
- 검증 지표는 확정 관리 진입률, 클릭률, 재선택 사용률, 최종 선택 완료율, 이탈률을 중심으로 본다.

**보조 문장**
- 시스템의 성공 기준은 `희귀 결과가 실패로 끝나지 않게 하면서도, 직접 획득 가치가 더 높게 유지되는가`이다.

**이미지 필요**
- 확정 관리 리스트 누적 상태 / 완료 팝업 / 영웅 인벤토리 중 1~2컷
